Redis入门笔记

发表于 2020-12-14 分类于 Redis 热度： Valine：
本文字数： 8.6k 阅读时长 ≈ 14 分钟

Redis（Remote Dictionary Server）是一种NoSQL型数据库，它不使用表来存储数据，而是使用key-value的形式，将数据存储在内存中，因此不存储海量数据，但是查询效率高。

《阿里巴巴中文架构》

传统RDBMS和NoSQL的区别

传统的RDBMS

  • 结构化组织
  • SQL
  • 数据和关系都存在单一的表中
  • 严格的一致性
  • 基础的事务

NoSQL

  • 不仅仅是数据
  • 没有固定的查询语言
  • 键值对存储，列存储，文档存储，图形数据库（社交关系）
  • 最终一致性
  • CAP定理和BASE（异地多活）
  • 高性能，高可用，高可扩

敏捷开发，极限编程

KV键值对一般使用redis，文档一般使用MongoDB，图像数据库Neo4J，infoGrid

MongoDB是一个基于分布式文件存储的数据库，C++编写，主要用于处理大量的文档。是一个介于关系型数据库和非关系型数据库中间产品，是非关系型数据库中功能最丰富，最像关系型数据库的。

Redis的基本知识

Redis-benchmark：原理测试工具，用来测试Redis的高并发性能。（菜鸟教程查看参数）

在Redis客户端可以使用ping命令查看Redis服务是否正常运行，如果正常则返会pong。

查看Redis服务器的统计信息，在Redis客户端运行：info。可以使用info [需要查看的信息]来查看指定的信息，例如：使用info Replication命令查看集群信息。

Redis的数据库实例：类似于MySQL中的数据库实例，但是区别是MySQL的数据库实例可以自己创建自己维护，Redis中的数据库实例只能由Redis创建维护，开发人员不能修改和创建数据库实例。

默认情况下，Redis默认会创建16个数据库实例，采用编号进行区分（0-15），使用时通过编号来使用，可以通过配置文件指定创建的数据库实例个数。

默认情况下，链接的是编号为0的数据库，可以使用select [数据库实例编号]切换数据库实例。

查看当前数据库中实例的多少(key的数量)：dbsize

查看当前数据库中的所有key：keys *

清空当前数据库实例：flushdb

清空所有数据库实例：flushall

查看Redis中的配置信息：config get *(所有的)，使用config get [参数名称]查看指定信息。

数据类型及主要特性

Redis提供的数据类型主要分为5种自有类型和一种自定义类型，这5种自有类型包括：String类型、哈希类型、列表类型、集合类型和顺序集合Zset类型。

String类型：

它是一个二进制安全的字符串，意味着它不仅能够存储字符串、还能存储图片、视频等多种类型, 最大长度支持512M。

哈希类型：

该类型是由field和关联的value组成的map。其中，field和value都是字符串类型的。

列表类型：

该类型是一个插入顺序排序的字符串元素集合，基于双链表实现。

集合类型：

Set类型是一种无顺序集合, 它和List类型最大的区别是：集合中的元素没有顺序, 且元素是唯一的。

Set类型主要应用于：在某些场景，如社交场景中，通过交集、并集和差集运算，通过Set类型可以非常方便地查找共同好友、共同关注和共同偏好等社交关系

顺序集合类型：

ZSet是一种有序集合类型，每个元素都会关联一个double类型的分数权值，通过这个权值来为集合中的成员进行从小到大的排序。与Set类型一样，其底层也是通过哈希表实现的。

Redis 关于key的操作命令

关于key的操作命令：

查询：

keys [需要查询的key]查询key，可以使用通配符。例如：keys *：查询当前实例中所有的key，keys k*查询所有以k开头的key，keys k*y….，keys k?y…，keys k[a,e]y匹配为kay/key的key。

判断key在数据库中是否存在

exists [需要查询的key值]，如果存在返回1，不存在返回0，exists [k1,k2,k3,...]：判断多个key是否存在，返回值是key存在的数量。

移动指定key到指定的数据库实例

move [指定的key] [数据库实例编号]

查看指定key的剩余生存时间

ttl [指定key]，返回-1表示没有设置最大生命时间，-2表示查询的key不存在

给key设置最大生存时间

expire [指定key] [生存时间 s]

查看指定key的数据类型

type [指定key]

重命名key

rename [指定key] [新名称]

删除key

删除指定key：del [指定key]，删除多个key：del [k1 k2 ...]，返回值是实际删除的key数量，flushall，删除当前数据库实例中所有key。

String

添加数据到 Redis 中set [键] [值]，例如set name zhangsan，使用get [键]获取。set设置值时，如果键已存在，则会进行覆盖。

添加一个string类型的数据到 Redis 中，同时设置他的失效时间：setex [键] [存活时间] [值]

添加一个String类型的数据到 Redis 中，不覆盖之前已存在的key：setnx [键] [值]，返回0表示key值已存在。

批量添加数据到 Redis 中：mset [k1] [v1] [k2] [v2] .....

批量添加数据到 Redis 中，所有key不存在时成功，否则放弃：msetnx [k1] [v1] [k2] [v2] ...

追加字符：append [指定键] [值]，返回追加之后值的长度。key不存在就会创建一个新的表达式。

对字符串类型数值进行加运算：incrby [指定key] [offset变化量]，返回计算之后的值。

对字符串类型数值进行减运算：decrby [指定value] [offset变化量]。

对字符串类型数值进行自增运算：incr [指定键]。自动进行加1，返回计算之后的结果。如果key不存在则初始化一个对应键值，并对值进行加1操作。如果键对应的值不是数值则报错。

对字符串类型数值进行自减运算：decr [指定键]，同incr。

覆盖已有字符串，setrange [指定键] [开始字符下标] [替换值]。返回值为覆盖之后字符串的长度。

从Redis中批量获取：mget [k1] [k2] [k3] ...，若不存在则返回(nil)

获取指定键对应值的长度：strlen [指定键]

获取指定字符串中指定字段：getrange [指定键] [开始字符下标] [结束字符下标]，返回截取之后的字符。闭区间，下标可以为负数，表示自右向左开始，-1开始。

List

将一个或者多个值依次插入到列表的表头：lpush [key] [v1] [v2] ...，返回值为列表中值的个数。例如：lpush test 1 2 3得到的列表为3 2 1。

将一个或者多个值依次插入到列表的表尾：rpush [key] [v1] [v2] ...，返回值为列表中值的个数。例如：lpush test 1 2 3得到的列表为1 2 3。

获取指定列表中的指定元素：lrange [指定键] [开始下标] [结束下标]。

获取指定列表中指定下标的元素：lindex [指定键] [指定下标]。

获取指定列表的长度：llen [指定键]

删除指定列表中的表头元素：lpop [指定键]，返回值为移除的元素

删除指定列表中的表尾元素：rpop [指定键]，返回值为移除的元素

删除指定列表中的指定元素：lrem [指定键] [count] [指定值]，count > 0，从表头的左侧移除count个数据，如果count < 0从列表右侧移除count个数据，如果count = 0，则表示全部删除

Set

向指定键中添加值sadd [指定集合] [元素] ...，例如：sadd test 0 1 2 3，如果元素已存在，则忽略放入操作。

获取指定set集合中的元素：smembers [指定集合]

获取指定set集合中元素的个数：scard [指定集合]

判断指定元素是否指定set集合中的元素：sismember [指定集合] [指定元素]

随机获取指定集合中的一个元素：srandmember [指定集合] [count]，count表示元素个数，默认为1，count > 0，表示随机获取多个元素不重复，count < 0，随机获取的多个元素可以重复 。

从集合A移动一个元素到集合B：smove [源集合] [目标集合] [指定元素]

删除集合中的指定元素：srem [指定集合] [指定元素] ...

随机删除元素，并返回：spop [指定键] [元素个数]，默认1个

求交集sinter [集合1] [集合2] ...

求交集并将交集保存到一个集合中：sinter [目标集合] [集合1] [集合2] ...

求并集：sunion [集合1] [集合2] ...

求并集并将并集保存到一个集合中：sunionstore [目标集合] [集合1] [集合2] ...

求差集：sdiff [集合1] [集合2]

求差集并将差集保存到一个集合中：sdiff [目标集合] [集合1] [集合2] ...

Hash

添加字段，元素到指定集合：hset [指定集合] [字段] [值] ...

添加多个字段，元素到指定集合：hmset [指定集合] [字段] [值] ...

获取指定集合中指定资源的元素：hget [指定集合] [指定字段]

获取指定集合中的多个指定元素：hmget[指定集合] [指定字段] ...

获取指定hash集合的长度：hlen [指定集合]

获取指定集合中的所有字段：hkeys [指定集合]

获取指定集合中所有的值：hvals [指定集合]

获取指定集合中所有的键值对：hgetall [指定集合]

判断集合中字段是否存在：hexists [指定集合] [指定字段]，返回1表示存在，返回0表示不存在。

对指定集合中指定字段的值进行加法运算：hincrby [指定集合] [指定字段] [count]，只能对值为数字的数据进行操作

删除指定集合中指定字段：hdel [指定集合] [指定字段] ...

Zset

添加指定元素和元素对应分数到指定集合：zadd [指定集合] [元素的分数] [指定元素] ...，如果添加的元素已经存在，则覆盖分数。

获取指定元素在指定集合中的名次：zrank [指定集合] [指定元素]

获取指定元素降序排列时的排名：zrevrank [指定集合] [指定元素]

获取指定元素的分数：zscore [指定集合] [指定元素]

获取指定集合中元素的个数：zcard [指定集合]

获取指定分数区间中元素的数量：zcount [指定集合] [分数1] [分数2]

获取指定集合中，指定名次内的元素：zrange [指定集合] [起始名次] [结束名次] [wirhsocres]，withscores参数表示同时显示分数。按照反方向：zrevrange...

获取指定集合指定分数区间中的所有元素：zrangebyscore [指定集合] min max [withscores] limit offset count，withscores参数表示同时显示分数。闭区间

删除指定集合中的元素：zrem [指定集合] [元素1] [元素2] ...

删除指定集合中指定分数范围的元素：zremrangebyscore [指定集合] [分数1] [分数2]

删除指定集合中指定名次氛围的元素：zremrangebyrank [指定集合] [名次1] [名次2]

求交集并将交集保存到另一个集合中：zinterstore [目标集合] numkeys [集合1] [集合2] ... [WEIGHTS weight1 [weight2...]] [AGGREGATE SUM|MIN|MAX]

key1，key2的权值分别是weight1，weight2，聚合方法用 sum|min|max

注意：weights，aggregate如何理解？

答：如果有交集，交集元素又有score，score怎么处理？aggregate num->score相加，min最小score，max最大score，另外可以通过weights设置不同的key的权重，交集时 score*weight

Redis 持久化

Redis提供了持久化功能：RDB和AOF(Append Only File)持久化方案

RDB

RDB(Redis DateBase)是Redis默认的持久化方案。在指定的时间间隔内执行指定次数的写操作，则会将内存中的数据写入到磁盘，即在指定的目录下生成一个dump.rdb文件。Redis重启时会通过加载该dump.rdb文件恢复数据。

原理：Redis会复制一个与当前进程一样的进程，新进程的所有数据（变量、环境变量、程序计数器等）数值都和原进程一致，但是是一个全新的进程，并作为原进程的子进程来进行持久化。整个过程中，主进程时不进行任何IO操作的，这就确保了极高的性能。

配置

save <seconds> <changes>：配置复合的快照触发条件，即Redis在seconds秒内key改变changes次，Redis把快照内的数据保存到磁盘一次。默认的策略是：

  • 一分钟内改变一万次
  • 五分钟内改变了10次
  • 15分钟内改变了1次

如果需要禁用Redis的持久化功能，则把所有的 save 配置都注释掉。

stop-writes-on-bgsave-error：当bgsave快照操作出错时停止写数据到磁盘，这样能保证内存数据和磁盘数据一致，但是如果不在乎这种一致性，这里配置no

rdbcompression：设置对于存储到磁盘中快照是否进行压缩，设置yes时，会采用LZF算法进行压缩，可以设置no来关闭这一功能。

rdbchecksum：在存储快照之后，还可以让Redis通过CRC64算法来进行数据校验，这会消耗一定的性能，可以设置no来进行关闭

dbfilename：Redis持久化数据生成的文件名，默认是dump.rdb，也可以自己配置

dir：Redis持久化数据生成文件保存的目录，默认是./，即Redis的启动目录，可以自己指定

AOF

采用操作日志来记录进行的每一次写操作，效率较低，Redis默认不开启

appendonly：配置是否开启AOF，yes表示开启，no表示关闭，默认为no

appendfilename：AOF保存文件名

appendfsync：AOF 异步持久化策略

  • always：同步持久化，每次发生数据变化会立刻写入到磁盘中，性能比较差但是数据完整性比较好
  • everysec：每秒异步记录一次，默认值
  • no：不即时同步，由操作系统决定何时同步

no-appendfsync-on-rewrite0：重写时是否可以运用appendsync，默认是no，可以保证数据的安全性

auto-aof-rewrite-percentage：设置重写的基准百分比

auto-aof-rewrite-min-size：设置重写的基准值

Redis的事务

事务：把一组数据库的事务放在一起执行，保证操作的原子性，要么同时成功，要么同时失败。

Redis的事务，把一组Redis命令放在一起，命令进行排序依次执行，只能保证部分原子性，严重错误保证原子性（压入事务队列时错误），一般错误不保证原子性（单条语句执行失败，不会导致事务失败）。

命令

multi：用来标记一个事务的开始。会将之后的命令压入队列，等待执行

exec：用来执行事务队列中的所有命令

discard：放弃已压入队列中命令，并且结束事务

watch：用来监控某一个或某几个key，此key对应的value一旦发生改变，则放弃本事务。类比乐观锁

unwatch：放弃监控所有键对事务的影响。

消息发布与订阅

Redis多客户端之间的通信，通过订阅频道，消息的发布者往频道上发布消息，这样所有订阅者都可以接收到消息。

publish [频道名] [消息]：将消息发送到指定频道

subscribe [频道名] ...：订阅一个或多个频道的消息

psubscribe [频道名]：订阅一个或多个频道的消息，频道名允许使用通配符

搭建Redis集群

Redis的主从复制：写操作的数据库称为主库，读操作的数据库称为从库master/slave

一主二从

redis-cli -h 127.0.0.1 -p 6379：连接指定ip和端口号的Redis服务

info replication：查看每台Redis客户端在集群中的角色 ——> 主机：role:master 从机：role:slave ，已连接从机数量：connected_slave：1

设置指定主机的从机：slaveof [指定ip] [指定端口]

全量复制：一旦主从关系确定，会自动把主库上已有数据同步复制到从库

增量复制：主库写数据会自动同步到从库

如果在从机上进行写操作：会报错You can't write against a read only replica

主机宕机时从机不会更改主从状态，只会修改主机连接状态：master_link_status:down，从机还是可以进行读操作。一旦主机恢复，从机会自动连接主机：master_link_status: up。

从机宕机时主机对主从机没有任何影响，恢复之后需要执行slaveof [指定ip] [指定端口]重新设置主从关系。

从机上位：假如主机宕机，使用某从机代替原主机，将指定从机断开原来的主从关系：slaveof no one，让其它从机与该从机设置主从关系。

从机下面也可以有从机。

这个方法虽然减轻了一台主机的压力，但是增加了服务间的延迟时间。

哨兵模式

主机宕机的时候自动进行从机上位。

部署哨兵监控主机，哨兵模式配置文件：redis安装目录/sentinel.conf

可以自定义配置文件：redis_sentinel.conf

启动哨兵服务：redis-sentinel redis_sentinel.conf

使用哨兵模式，一旦之前的主机恢复之后，哨兵程序会自动将该服务从属于新的主机成为从机
