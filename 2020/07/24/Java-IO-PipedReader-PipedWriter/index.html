PipedWriter 和 PipedReader 简介

PipedWriter是字符管道输出流，它继承于Writer。

PipedReader是字符管道输入流，它继承于Reader。

PipedWriter和PipedReader的作用是可以通过管道进行线程间的通讯。在使用管道通信时，必须将PipedWriter和PipedReader配套使用。

由2和3的分析可知，Sender要往管道写入1046个字符。其中，前1024个字符(缓冲区容量是1024)能正常写入，并且每写入一个就读取一个。当写入1025个字符时，依然是依次的调用PipedWriter中的write()；然后，write()中调用PipedReader中的receive()；在PipedReader中，最终又会调用到receive(int c)函数。 而此时，管道输入流已经被关闭，也就是closedByReader为true，所以抛出throw new IOException("Pipe closed")。
